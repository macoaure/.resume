# i18n Context Rule

## Translation Key Pattern

In this project, **the key is the fallback language (Portuguese Brazil) and the value is the translated version**.

### Supported Locales:
- `pt-br` - Portuguese (Brazil) - Fallback language
- `en-us` - English (United States)

### Example:
```json
{
  "Currículo": "Resume",
  "Bem-vindo ao meu currículo": "Welcome to my resume",
  "Início": "Home"
}
```

### Usage in Templates:
```vue
<h1>{{ $t('Currículo') }}</h1>
<p>{{ $t('Bem-vindo ao meu currículo') }}</p>
```

### File Structure:
- `src/locales/pt_BR.json` - Portuguese (Brazil) - Keys and fallback values
- `src/locales/en_US.json` - English (United States) - Portuguese keys mapped to English values

### Benefits:
- Keys are in the fallback language (Portuguese Brazil)
- Self-documenting: keys are human-readable
- Automatic fallback to Portuguese if translation is missing
- Easier to maintain for Portuguese-first projects

### When Adding New Translations:
1. Write the Portuguese text as the key
2. Provide translations in `en_US.json` for each locale
3. Example: Add to both files with Portuguese key and translated value
